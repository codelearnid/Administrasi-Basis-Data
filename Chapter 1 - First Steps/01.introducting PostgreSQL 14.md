**Mengenal PostgreSQL 14**

PostgreSQL adalah server basis data berbasis SQL yang canggih dan dapat digunakan di berbagai platform. Salah satu keunggulan utama PostgreSQL adalah bersifat *open source* (sumber terbuka), artinya Anda bebas menginstal, menggunakan, dan menyebarkan PostgreSQL tanpa harus membayar biaya atau royalti kepada siapa pun. Selain itu, PostgreSQL dikenal sebagai basis data yang bisa berjalan lama tanpa perlu sering dimatikan, dan umumnya membutuhkan sedikit atau bahkan tanpa perawatan sama sekali. Secara keseluruhan, PostgreSQL memiliki biaya kepemilikan yang sangat rendah.

PostgreSQL juga terkenal karena memiliki banyak fitur canggih, yang dikembangkan selama lebih dari 30 tahun melalui proses pengembangan dan penyempurnaan yang terus-menerus. Awalnya dibuat oleh Kelompok Penelitian Basis Data di University of California, Berkeley, kini PostgreSQL dikembangkan dan dipelihara oleh komunitas besar para pengembang dan kontributor. Banyak dari mereka bekerja penuh waktu dalam bidang yang terkait dengan PostgreSQL, seperti perancang sistem, pengembang perangkat lunak, administrator basis data, dan pelatih. Sebagian kecil dari mereka bekerja di perusahaan yang khusus menyediakan dukungan untuk PostgreSQL.

Tidak ada satu perusahaan pun yang memiliki hak penuh atas PostgreSQL, dan Anda tidak diwajibkan (bahkan tidak diminta) untuk mendaftarkan penggunaan PostgreSQL yang Anda lakukan.

Berikut adalah beberapa fitur utama PostgreSQL:

- **Kompatibilitas yang sangat baik dengan standar SQL**, hingga standar SQL:2016.  
- **Arsitektur client-server**, di mana klien bisa terhubung ke server basis data dari jarak jauh.  
- **Dirancang untuk mendukung banyak pengguna secara bersamaan**, sehingga pengguna yang membaca data tidak mengganggu pengguna yang sedang menulis/mengubah data.  
- **Sangat fleksibel dan bisa dikembangkan**, cocok untuk berbagai jenis aplikasi.  
- **Skalabilitas dan kinerja yang baik**, dengan banyak opsi penyesuaian (tuning) agar bekerja lebih cepat.  
- **Mendukung berbagai model data**, seperti:  
  - Data relasional (tabel biasa),  
  - Data pasca-relasional (misalnya array atau relasi bersarang melalui tipe rekaman),  
  - Data dokumen (seperti format JSON dan XML),  
  - Dan data berbentuk pasangan kunci-nilai (key/value).  

Dengan semua fitur ini, PostgreSQL menjadi pilihan yang kuat dan andal untuk berbagai kebutuhan penyimpanan dan pengelolaan data.

**Apa yang Membuat PostgreSQL Berbeda?**

Proyek PostgreSQL fokus pada beberapa tujuan utama berikut:

- Membuat perangkat lunak yang **kuat dan berkualitas tinggi**, dengan kode yang rapi dan mudah dimengerti.  
- Mudah dikelola, baik untuk penggunaan kecil (embedded) maupun skala besar (enterprise), tanpa perlu perawatan berlebihan.  
- Mendukung **standar SQL**, bisa bekerja sama dengan sistem lain, dan kompatibel dengan berbagai aplikasi.  
- Memiliki kinerja yang cepat, aman, dan siap digunakan hampir tanpa henti (tingkat ketersediaan tinggi).

Banyak orang terkejut mengetahui bahwa fitur-fitur PostgreSQL lebih mirip dengan Oracle atau SQL Server daripada MySQL. Satu-satunya kesamaan antara MySQL dan PostgreSQL adalah keduanya *open source*. Selebihnya, fitur dan pendekatannya sangat berbeda.

Salah satu fitur unggulan Oracle sejak Oracle 7 adalah *snapshot isolation*, yaitu saat pengguna yang membaca data tidak mengganggu pengguna yang sedang menulis, dan sebaliknya. Tapi mungkin Anda akan terkejut mengetahui bahwa **PostgreSQL justru yang pertama kali dirancang dengan fitur ini**. Di PostgreSQL, fitur ini disebut **MVCC (Multiversion Concurrency Control)**, dan akan kita bahas lebih dalam nanti.

PostgreSQL adalah sistem manajemen basis data yang bisa digunakan untuk berbagai keperluan. Anda bisa menentukan sendiri bagaimana basis data Anda ingin dikelola. PostgreSQL memberi banyak pilihan cara kerja:

- Anda bisa menggunakan model basis data terstruktur (normalisasi), ditambah fitur seperti *array* dan tipe data rekaman.  
- Atau, Anda bisa menggunakan skema yang sangat fleksibel dengan bantuan format **JSONB** dan ekstensi bernama **hstore**.  
- Anda juga bisa membuat fungsi sendiri di sisi server, menggunakan lebih dari satu lusin bahasa pemrograman seperti Python, Perl, JavaScript, dan lainnya.

**1. Contoh Penggunaan JSONB**

**JSONB** adalah tipe data di PostgreSQL yang menyimpan data dalam format **JSON (JavaScript Object Notation)** secara biner, sehingga cepat diakses dan bisa diindeks.

📌 Contoh Kasus:
Kita ingin menyimpan data profil pengguna yang fleksibel, misalnya hobi, alamat, atau pengaturan notifikasi — tanpa harus membuat kolom baru setiap kali ada data tambahan.

💡 Buat tabel dengan kolom JSONB:
```sql
CREATE TABLE pengguna (
    id serial PRIMARY KEY,
    nama text,
    data_lain jsonb
);
```

💡 Masukkan data:
```sql
INSERT INTO pengguna (nama, data_lain) VALUES
('Andi', '{
    "hobi": ["membaca", "bersepeda"],
    "alamat": "Jakarta",
    "notifikasi": true,
    "umur": 28
}');
```

💡 Ambil data tertentu dari JSONB:
```sql
-- Cari pengguna yang hobi-nya bersepeda
SELECT nama FROM pengguna 
WHERE data_lain->'hobi' ? 'bersepeda';
```

> Hasil: `Andi`

---

**2. Contoh Penggunaan hstore**

**hstore** adalah ekstensi PostgreSQL yang memungkinkan Anda menyimpan pasangan **kunci-nilai (key-value)** seperti `nama_kolom => nilai`, mirip dengan dictionary atau map.

💡 Aktifkan ekstensi hstore dulu:
```sql
CREATE EXTENSION IF NOT EXISTS hstore;
```

💡 Buat tabel dengan kolom hstore:
```sql
CREATE TABLE produk (
    id serial PRIMARY KEY,
    nama text,
    atribut hstore
);
```

💡 Masukkan data:
```sql
INSERT INTO produk (nama, atribut) VALUES
('Laptop', '("warna" => "hitam", "ram" => "16GB", "tipe" => "gaming")'),
('Mouse', '("warna" => "putih", "baterai" => "rechargeable")');
```

💡 Ambil data dari hstore:
```sql
-- Cari produk yang warnanya hitam
SELECT nama FROM produk 
WHERE atribut->'warna' = 'hitam';
```

> Hasil: `Laptop`

```sql
-- Tampilkan semua atribut laptop
SELECT atribut FROM produk WHERE nama = 'Laptop';
```

> Hasil: `"warna"=>"hitam", "ram"=>"16GB", "tipe"=>"gaming"`

---

🔎 Perbedaan Singkat JSONB vs hstore

| Fitur | JSONB | hstore |
|------|-------|--------|
| Format | JSON (struktur terstruktur) | Kunci-nilai sederhana |
| Tipe data | Mendukung string, angka, array, boolean, objek | Hanya string untuk kunci dan nilai |
| Indeks | Bisa diindeks dengan GIN (sangat cepat) | Bisa diindeks |
| Fleksibilitas | Sangat fleksibel, cocok untuk data kompleks | Sederhana, cocok untuk metadata sederhana |

---

✅ Kapan Harus Pakai Apa?

- Gunakan **JSONB** jika:  
  Anda butuh menyimpan data seperti profil, pengaturan, atau dokumen dengan struktur bersarang (nested), dan ingin fleksibilitas seperti NoSQL.

- Gunakan **hstore** jika:  
  Anda hanya butuh menyimpan pasangan kunci-nilai sederhana, seperti tag, metadata, atau konfigurasi cepat.

Dengan keduanya, PostgreSQL bisa berfungsi seperti **database NoSQL**, sekaligus tetap menjadi **database relasional** yang kuat. Itulah yang membuatnya sangat fleksibel!

PostgreSQL juga sangat bisa dikembangkan (ekstensibel). Artinya, Anda bisa:

- Menambahkan tipe data baru,  
- Membuat operator khusus,  
- Menambah jenis indeks baru,  
- Bahkan menambah bahasa pemrograman fungsional sendiri.  

Anda bahkan bisa mengubah cara kerja sistem menggunakan *plugin*, mengganti cara perintah dijalankan, atau menambah *optimizer* kueri baru.

Semua fitur ini memberi banyak kebebasan bagi arsitek perangkat lunak dalam membangun aplikasi. Jika ada masalah, ada banyak cara untuk mengatasinya, baik saat membangun maupun saat merawat aplikasi dalam jangka panjang. Sayangnya, buku ini tidak bisa membahas semua fitur keren untuk pengembang, karena fokus buku ini adalah **administrasi, perawatan, dan pencadangan (backup) basis data**.

Dulu, saat PostgreSQL masih sekadar proyek riset, fokusnya hanya pada fitur-fitur baru yang menarik. Namun selama 20 tahun terakhir, hampir semua kode telah ditulis ulang dan diperbaiki, sehingga kini PostgreSQL menjadi salah satu server perangkat lunak terbesar dan paling stabil yang bisa digunakan secara operasional.

**Siapa yang menggunakan PostgreSQL?**  
Banyak perusahaan dan lembaga besar, seperti Apple, BASF, Genentech, Heroku, IMDB, Skype, McAfee, NTT, Badan Meteorologi Inggris (UK Met Office), dan Layanan Cuaca Nasional AS (US National Weather Service). Pada awal 2010, PostgreSQL sudah diunduh lebih dari 1 juta kali per tahun. Menurut data yang diajukan ke Komisi Eropa, *"PostgreSQL dianggap oleh banyak pengguna sebagai alternatif yang bisa diandalkan."* Sejak itu, popularitasnya terus meningkat.

Ada satu hal penting yang perlu diketahui:  
Dulu, PostgreSQL bernama **Postgres**. Karena itu, banyak bagian sistem masih menggunakan nama tersebut. Misalnya, basis data bawaan (default) bernama **postgres**, dan perangkat lunak sering diinstal menggunakan akun pengguna bernama **Postgres**. Oleh karena itu, banyak orang menyebut PostgreSQL dengan sebutan **Postgres**, dan kedua nama ini sering digunakan secara bergantian.

- **Pronunciation (cara mengucapkan):**  
  - *PostgreSQL* diucapkan **"post-grez-kyu-el"**  
  - *Postgres* diucapkan **"post-grez"**  
  - Hindari menyebutnya *Postgre* — selain sulit diucapkan, ini bisa membingungkan. Cukup gunakan dua nama: PostgreSQL atau Postgres.

Berikut penjelasan lebih rinci tentang beberapa aspek utama:

---

**Ketangguhan (Robustness)**  
PostgreSQL adalah perangkat lunak yang sangat andal, didukung oleh pengujian menyeluruh untuk fitur dan kinerja saat digunakan banyak orang sekaligus. Secara bawaan, PostgreSQL menjamin data yang ditulis ke disk benar-benar tersimpan dengan aman. Para pengembang sangat serius menjaga agar data tidak hilang.

Semua operasi di PostgreSQL dilakukan dalam **transaksi**, dan dilindungi oleh *log transaksi* yang bisa melakukan pemulihan otomatis jika terjadi kegagalan sistem.

Anda juga bisa membuat basis data dengan *checksum* (pemeriksaan integritas data) untuk membantu mendeteksi kerusakan pada perangkat keras. Ada berbagai cara untuk membuat cadangan data, termasuk **Point-in-Time Recovery (PITR)** yang memungkinkan pemulihan data secara sangat rinci. Banyak alat diagnosa juga tersedia untuk membantu menemukan masalah.

Replikasi basis data (menyalin data ke server lain) didukung secara bawaan. Dengan konfigurasi yang tepat, replikasi sinkron bisa memberikan ketersediaan hingga **99,999% (5 nines)** atau bahkan lebih tinggi jika ditambah redundansi.

---

**Keamanan (Security)**  
Akses ke PostgreSQL bisa dikontrol lewat aturan berbasis host (komputer mana yang boleh terhubung). Sistem autentikasi (verifikasi identitas) sangat fleksibel dan bisa disesuaikan, sehingga mudah diintegrasikan dengan sistem keamanan eksternal.

Fitur keamanan terbaru seperti **SCRAM (Salted Challenge Response Authentication Mechanism)** memberikan perlindungan kuat hingga **256-bit**, sehingga sangat sulit ditembus.

---

Singkatnya, PostgreSQL bukan sekadar basis data biasa — ia andal, aman, fleksibel, dan dirancang untuk digunakan dalam berbagai skenario, dari aplikasi kecil hingga sistem besar berskala global.

**Akses SSL Penuh dan Keamanan Data**

PostgreSQL mendukung **enkripsi SSL secara penuh** untuk koneksi pengguna maupun replikasi data (penyalinan data ke server lain). Artinya, semua data yang dikirim aman dari penyadapan. Selain itu, PostgreSQL menyediakan **pustaka fungsi kriptografi** yang lengkap, sehingga pengguna bisa mengamankan data mereka langsung di dalam basis data.

**Hak Akses dan Keamanan Tingkat Lanjut**

PostgreSQL menggunakan sistem **hak akses berbasis peran (role-based access)**, yang memungkinkan Anda mengatur siapa yang bisa membaca, menulis, atau mengubah data, tergantung pada jenis perintah.

Fitur penting lainnya adalah **Keamanan Tingkat Baris (Row-Level Security / RLS)**, yang memungkinkan Anda mengatur agar pengguna hanya bisa melihat data tertentu sesuai izin mereka. Fitur ini sangat berguna untuk aplikasi yang membutuhkan privasi tinggi, seperti sistem medis atau militer.

Fungsi (functions) bisa dijalankan dengan izin si pembuat (definer), dan *view* (tampilan data) bisa dibuat dengan pengaman tambahan (*security barrier*) agar aturan keamanan diterapkan sebelum data diproses.

Tim keamanan aktif selalu memantau seluruh bagian PostgreSQL. Jika ditemukan celah keamanan, informasinya langsung dipublikasikan di:  
[http://www.postgresql.org/support/security/](http://www.postgresql.org/support/security/)

---

**Mudah Digunakan**

Dokumentasi PostgreSQL sangat **lengkap, jelas, dan akurat**, karena setiap perubahan kode harus disertai pembaruan dokumentasi. Setiap rilis baru biasanya mencakup ratusan perbaikan kecil dari masukan pengguna, sehingga pengalaman menggunakan PostgreSQL terus membaik dari waktu ke waktu.

PostgreSQL bekerja dengan baik baik di komputer kecil maupun besar, dan bisa berjalan di berbagai sistem operasi (Windows, Linux, macOS, dll).

Tersedia **konektor (driver)** untuk hampir semua bahasa pemrograman dan lingkungan pengembangan, jadi Anda bebas memilih teknologi apa pun sekarang atau di masa depan.

PostgreSQL sangat **setia pada standar SQL**, tidak ada perilaku aneh seperti data yang diam-diam dipotong tanpa peringatan.

Untuk data teks, PostgreSQL menggunakan satu tipe data yang bisa menyimpan mulai dari **1 byte hingga 1 gigabyte**. Sistem penyimpanannya sangat efisien: data kecil disimpan secara hemat, sementara data besar dikelola dan dikompresi secara otomatis.

PostgreSQL juga berusaha **meminimalkan jumlah pengaturan (konfigurasi)** yang harus diatur manual. Setiap rilis baru berusaha meningkatkan kemampuan *auto-tune* (penyesuaian otomatis) agar sistem bekerja optimal tanpa perlu banyak campur tangan.

---

**Kemampuan Dikembangkan (Extensibility)**

PostgreSQL dirancang agar sangat **mudah dikembangkan**. Anda bisa menambahkan fitur baru dengan perintah sederhana:  
`CREATE EXTENSION`, yang akan mengatur versi, ketergantungan, dan konfigurasi secara otomatis.

Anda bisa membuat:
- Tipe data sendiri  
- Operator khusus (misalnya `@>` atau `#=`)  
- Jenis indeks baru  
- Fungsi dalam berbagai bahasa (Python, Perl, dll)  
- Bahkan bahasa pemrograman server sendiri

Ada banyak ekstensi siap pakai, seperti **PostGIS**, yang memberikan fitur **Sistem Informasi Geografis (GIS)** kelas dunia — sangat berguna untuk aplikasi peta, pelacakan lokasi, dan analisis spasial.

---

**Kinerja dan Kemampuan Banyak Pengguna (Concurrency)**

PostgreSQL 14 mampu melakukan **lebih dari 1 juta pembacaan per detik** pada server dengan 4 prosesor, dan lebih dari **30.000 transaksi penulisan per detik** dengan jaminan data tidak hilang (durability), tergantung pada perangkat keras yang digunakan. Dengan perangkat keras canggih, kinerjanya bisa jauh lebih tinggi.

PostgreSQL memiliki **optimizer (pengoptimal kueri) yang canggih**, yang memilih cara terbaik menjalankan perintah dengan mempertimbangkan berbagai jenis *join* dan statistik data pengguna.

PostgreSQL menyediakan **jenis indeks terbanyak** dibanding basis data lain, dan mendukung semua tipe data secara penuh.

Fitur **MVCC (Multiversion Concurrency Control)** memungkinkan banyak pengguna membaca dan menulis data secara bersamaan tanpa saling mengganggu.

Gabungan semua fitur ini membuat PostgreSQL bisa menangani **campuran beban kerja**: mulai dari transaksi harian, pencarian kompleks, hingga analisis data berat — **semua dalam satu sistem**.

Artinya, Anda **tidak perlu memindahkan data** dari sistem produksi ke gudang data khusus hanya untuk menjalankan satu atau dua kueri. Ini menghemat waktu, sumber daya, dan risiko kesalahan.

Karena kemampuannya, PostgreSQL menjadi **pilihan utama** untuk sistem baru, dan juga **pilihan jangka panjang** yang sangat andal.

---

**Skalabilitas (Bisa Dikembangkan Sesuai Kebutuhan)**

PostgreSQL 14 bisa bekerja sangat baik pada satu server hingga **empat soket CPU**. Dengan bantuan *session pool*, PostgreSQL bisa menangani ratusan sesi aktif dan ribuan koneksi sekaligus.

Setiap tahun, kemampuan skalanya terus ditingkatkan.

Untuk skala lebih besar, PostgreSQL mendukung **skalabilitas baca (read scalability)** melalui fitur *hot standby* (server cadangan yang siap digunakan).

Sementara itu, **skalabilitas tulis (write scalability)** di banyak server sedang dalam pengembangan aktif. Dasarnya adalah **replikasi dua arah (Bi-directional Replication)**, yang akan dibahas di Bab 12.

---

**Dukungan untuk Model Data SQL dan NoSQL**

PostgreSQL sangat **setia pada standar SQL**, tapi tidak memaksa Anda menggunakan model tertentu. Anda bisa menggunakan berbagai model data **dalam satu basis data yang sama**.

Sebagai basis data relasional, Anda bisa memilih tingkat *denormalisasi* yang diinginkan:
- Dari model terstruktur ketat (Third Normal Form / 3NF)  
- Hingga model *star schema* yang lebih sederhana untuk analisis

PostgreSQL memperluas model relasional dengan dukungan untuk:
- Array (daftar data)  
- Tipe baris (row types)  
- Tipe rentang (range types), misalnya rentang tanggal

Anda juga bisa menggunakan PostgreSQL sebagai **basis data dokumen**, dengan tipe data seperti:
- Teks biasa  
- XML  
- JSONB (JSON dalam format biner yang cepat dan bisa diindeks)

Semua tipe dokumen ini didukung oleh **indeks khusus dokumen** dan fitur **pencarian teks lengkap (full-text search)**.

Untuk model **key-value (kunci-nilai)**, Anda bisa menggunakan ekstensi **hstore**, yang mirip dengan format JSON sederhana.

**Kesimpulan**

PostgreSQL bukan hanya sekadar database open source. Ia andal, aman, fleksibel, cepat, dan bisa digunakan untuk hampir semua jenis aplikasi — dari sistem kecil hingga infrastruktur perusahaan global. **Pendanaan untuk Penelitian dan Pengembangan**

PostgreSQL awalnya dikembangkan sebagai proyek penelitian di **Universitas California, Berkeley**, pada akhir tahun 1980-an dan awal 1990-an. Setelah itu, pengembangannya dilanjutkan oleh para relawan hingga akhir 1990-an. Kemudian, seorang pengembang profesional pertama kali terlibat, dan sejak saat itu semakin banyak perusahaan dan kelompok peneliti yang ikut berpartisipasi, serta mendukung para pengembang profesional.

Pengembangan PostgreSQL juga mendapat dana dari **NSF (National Science Foundation)**, sebuah lembaga di Amerika Serikat yang mendukung penelitian ilmiah. Selain itu, proyek ini juga menerima pendanaan dari **Program FP7 Uni Eropa**, dalam bentuk dua proyek besar:

1. **Proyek 4CaaST** – fokus pada komputasi awan (cloud computing).  
2. **Proyek AXLE** – fokus pada analisis data berskala besar.

**Proyek AXLE patut disebut khusus** karena berlangsung selama tiga tahun dan bertujuan memperkuat kemampuan PostgreSQL dalam **kecerdasan bisnis (business intelligence)**, terutama untuk database yang sangat besar. Proyek ini mencakup peningkatan dalam bidang:

- Keamanan dan privasi data  
- Integrasi dengan alat *data mining* (penambangan data)  
- Alat visualisasi data (menampilkan data dalam bentuk grafik atau dashboard)  
- Dukungan untuk perangkat keras baru  

Informasi lebih lanjut tentang proyek AXLE bisa dilihat di situs resminya:  
[https://axleproject.eu/](https://axleproject.eu/)

Selain dari lembaga penelitian dan pemerintah, pendanaan untuk pengembangan PostgreSQL juga berasal dari:

- **Pengguna langsung** yang mendanai pengembangan fitur-fitur tertentu.  
- **Perusahaan** yang menjual produk atau layanan berbasis PostgreSQL (seperti dukungan teknis, alat pengelolaan, atau versi perusahaan).

Banyak fitur baru ditambahkan secara rutin oleh perusahaan besar, seperti **EDB (EnterpriseDB)**, yang secara aktif berkontribusi pada pengembangan PostgreSQL.

Dengan dukungan dari berbagai pihak ini, PostgreSQL terus berkembang menjadi sistem basis data yang lebih kuat, aman, dan cocok untuk kebutuhan masa kini dan masa depan.
